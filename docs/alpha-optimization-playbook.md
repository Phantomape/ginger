# 个人量化交易系统 Alpha 优化思想指导

> 适用对象：当前 `ginger` 仓库  
> 目标：在不过拟合、不失控增加复杂度的前提下，提高系统的 `alpha` 与长期收入期望  
> 视角：最强大脑 + 最强金融分析师 + 最严实验科学

---

## 一、先说结论

这个个人量化交易系统**不是缺“更多策略”**，而是缺“更高质量的 alpha 归因与更稳定的资本配置”。

当前系统已经具备以下基础：

- 技术面信号引擎：`quant/signal_engine.py`
- 风控与目标位：`quant/risk_engine.py`
- 定仓与组合热度：`quant/portfolio_engine.py`
- 市场状态过滤：`quant/regime.py`
- 新闻过滤与 LLM 事件判断：`quant/filter.py` + `quant/llm_advisor.py`
- 历史回测与结果沉淀：`quant/backtester.py`

所以，下一阶段最能提升 `alpha` 和收入期望的，不是再堆新策略，而是围绕以下 4 个方向做精修：

1. **提升信号质量，而不是提升信号数量**
2. **提升 exit 质量，而不是只盯 entry 条件**
3. **让 LLM 从“模糊判断”升级为“结构化 alpha 过滤器”**
4. **让资金只流向最有单位风险回报的信号**

一句话概括：

> 未来的优化重点应该从”我还能再加什么规则”转向”我如何把已有信号筛成更高 EV 的少数机会，并把仓位压到最值钱的地方”。

> **当前系统状态快照（2026-04-18，124 交易日，观测样本）**
>
> | 指标 | 值 |
> |------|----|
> | `expected_value_score` | **0.4875**（= 19.98% 回报 × 2.44 日 Sharpe） |
> | 胜率 | 60% |
> | 最大回撤 | 4.59% |
> | 交易分布 | trend_long 13 笔 / breakout_long 17 笔 / **earnings_event_long 0 笔** |
> | 信号存活率 | **98.04%**（50/51） |
> | 跑赢 SPY/QQQ | +13.6 pp |
> | 收敛状态 | **8/8 标准通过（CONVERGED）** |
> | LLM 决策回放覆盖率 | **0%** ← 当前最大测量盲区 |
> | 新闻 veto 回放覆盖率 | 10.5% |
>
> **重要**：上述 `expected_value_score = 0.4875` 是在 **LLM veto 贡献完全未被回放**（覆盖率 0%）的条件下测得的。
> 生产中 LLM 每天都在否决信号；这些否决在回测中未被重放。
> 因此该值存在方向不确定的系统性偏差——既可能高估（若 LLM 历史上一直在帮），也可能低估（若 LLM 在误杀好信号）。
> 在 LLM 覆盖率修复前，不应将此值用于”LLM 有/无价值”的判断。
> 此外，这组数字只能作为**当前阶段的观测样本**，用于确定近端优化优先级；它们不构成对系统长期 alpha 质量的最终判定。

## 二、当前系统真正的 alpha 来源是什么

必须先明确这个问题，否则后续优化会发散。

这个系统当前并不是一个纯统计套利系统，也不是高频系统。  
它更接近一个**事件增强型中短线趋势 / 突破交易系统**。

当前 alpha 主要来自 3 个来源：

### 1. 趋势延续 alpha

系统通过：

- `价格 > 200MA`
- `20 日突破`
- `量价配合`
- `相对强弱/短期动量`

来捕捉机构资金推动下的趋势持续。

这类 alpha 的本质不是“预测基本面”，而是：

- 强势股更容易继续强
- 机构加仓具有惯性
- 市场对趋势的反应往往不是一次性完成

### 2. 波动突破 alpha

系统中的 breakout 逻辑，本质是在找：

- 波动放大
- 价格脱离整理区间
- 成交量确认

这类机会。

它赚的钱，来源不是“均值回归”，而是：

- 突破后跟风盘与机构延续买盘
- 市场对新价格区间的重新定价

### 3. 新闻/事件过滤 alpha

LLM 与新闻层最大的潜在价值不在“找更多票”，而在：

- **过滤掉伪信号**
- **识别真正重要的催化事件**
- **在同样技术图形中分辨哪一个更值得冒险**

这部分 alpha 更像“负向筛除 + 质量排序 alpha”，而不是独立生成 alpha。

换句话说：

> 当前系统最值得增强的，不是”更多 entry 模式”，而是”更强的信号质量判断与更高效的 capital allocation”。

**2026-04-18 实测修正：当前回测口径下，后段过滤并非高度挑剔**

过去文档和代码注释中曾估算”11 层过滤 × 70% 通过率 ≈ 2% 存活率”。
实测结果：存活率为 **98%（50/51 信号）**，与理论估算相差近 50 倍。

这意味着：
- 在**当前回测器可见的代码层后段过滤**中，额外杀伤有限；
- 但这**不等于生产中的总过滤强度也很低**，因为新闻 veto / LLM veto 尚未完整回放；
- 因此，当前更稳妥的判断不是“过滤器已无关紧要”，而是：  
  “应暂缓把增删过滤器作为第一优先级，先补齐生产/回测一致性，再决定过滤层是否真的过强或过弱。”

## 三、当前系统最可能限制收入期望的地方

从投研角度看，当前系统要最大化收入期望，最受限的不是“不会选股”，而是以下几个结构问题。

### 1. entry 已经不算太弱，但 exit 仍然可能吃掉 alpha

大多数个人交易系统会把 80% 的注意力放在“怎么进”，但真正决定收入期望的，往往是：

- 盈利后何时减仓
- 何时放盈
- 何时让利润奔跑
- 何时快速承认错误

当前仓库中，exit 体系已经有：

- hard stop
- ATR stop
- trailing stop
- signal target
- profit ladder
- time stop

这说明系统的瓶颈已经很可能转向：

- **是否止盈太早**
- **是否把大赢家切碎了**
- **是否在错误的波动环境下使用了相同退出逻辑**

如果未来只继续优化 entry，而不优化 exit，alpha 大概率会被泄漏。

### 2. LLM 目前更像 veto 层，还没有充分变成“排序层”

当前系统已经让 LLM 参与新闻理解，这是对的。  
但从最大化收入期望角度，仅仅做 veto 还不够。

真正更值钱的方向是让 LLM：

- 区分“强正催化 / 弱正催化 / 中性 / 负催化”
- 对事件做结构化评分
- 给已有 quant signal 做**质量排序**

也就是说：

- 代码负责“能不能做”
- LLM 负责“这几个都能做时，哪一个更值钱”

个人系统的资金和注意力都有限，**排序 alpha** 比“是否允许”更有价值。

### 3. 资金配置逻辑还有继续做强的空间

对个人账户来说，真正拉开收入差距的不是多做几笔，而是：

- 在最值得做的信号上多赚一点
- 在边际信号上少赔一点
- 在差环境里自动保守

当前系统已有定仓逻辑，但未来最值得加强的是：

- 不同信号质量分层的仓位差异
- 不同 regime 下的仓位差异
- 同日多个机会的优先级排序
- 与持仓周期、组合热度耦合的资本效率优化

也就是说，未来最重要的问题不是：

> “今天有没有信号？”

而是：

> “今天最值得我把风险预算押给谁？”

### 4. 评估体系还应更贴近“赚钱能力”而不是“规则完成度”

如果一个系统回测看起来很漂亮，但不能稳定提高净收入期望，那优化方向仍然可能偏。

未来应更多围绕以下问题评价优化是否有价值：

- 这个改动是否提升了 `expected_value_score`
- 是否提升了单位风险预算的回报
- 是否让最差交易簇变轻
- 是否让大赢家保留得更久
- 是否减少了“好信号被 veto / 差信号被放行”的错误

## 四、最值得优先做的策略优化方向

下面这些方向，是我认为当前系统最值得投入的 alpha 提升路径。

---

## 方向 A：先优化退出逻辑，而不是继续堆新的入场规则

### 核心思想

个人交易系统的 alpha，经常不是输在“不会买”，而是输在：

- 过早止盈
- 在真正进入趋势后不给仓位时间
- 用统一 target 去处理不同波动状态

### 应优先研究的问题

1. `SIGNAL_TARGET` 是否对强趋势票减仓过早  
2. `PROFIT_TARGET / PROFIT_LADDER` 是否在强 bull regime 中过度锁利  
3. `TRAILING_STOP` 是否足够贴近趋势，而不是太松或太紧  
4. 不同策略类型是否应该有不同 exit 结构  

### 推荐优化方向

- 对 `trend_long` 和 `breakout_long` 分开设计 exit 规则
- 在强 bull regime 中，减少固定 target 的主导地位，让 trailing 逻辑更重要
- 把“到目标位就减仓”改造成“到目标位后切换到更紧致的利润保护模式”
- 研究“大赢家保留率”而不是只看均值收益

### 为什么这最可能提升 alpha

因为个人系统最大的财富往往来自少数大赢家，而不是平均交易。  
如果 exit 结构能多保留几个真正的大趋势单，收入期望会显著上升。

---

## 方向 B：把 LLM 从 veto 层升级为结构化排序层

### 核心思想

LLM 最值钱的作用不是替代码做止损，而是做：

- 事件重要性识别
- 新闻强弱排序
- 真假催化区分
- 语义层的“这是不是值得押风险”

### 推荐 LLM 输出的结构化字段

每条新闻或每个候选信号，建议输出类似：

```json
{
  "event_type": "earnings_guidance_raise",
  "direction": "positive",
  "materiality": 0.88,
  "novelty": 0.76,
  "time_horizon": "short_term",
  "thesis_alignment": 0.81,
  "veto": false
}
```

然后代码再使用这些字段：

- `materiality` 高：提高排序优先级
- `thesis_alignment` 低：降低仓位或直接 veto
- `novelty` 低：避免把重复旧闻当新催化
- `direction` 负向且 `materiality` 高：直接 veto

### 为什么这会提升收入期望

因为现实里，技术图形相似的两个 breakout：

- 一个背后是强催化
- 一个背后是媒体噪音

如果系统能把这两者区分开，资金就会更集中在“同样技术面、但事件质量更高”的机会。

这比增加更多 entry pattern 更有价值。

---

## 方向 C：做“仓位分层”，而不是统一 1% 风险模板

### 核心思想

统一风险模板适合系统初期，但当信号质量有明显差异时，统一仓位会浪费 alpha。

最好的机会应该拿到更高质量的风险预算。

### 推荐分层方式

按以下组合维度给信号分层：

- `trade_quality_score`
- `expected_value_score` 相关代理字段
- LLM `materiality`
- 市场 regime
- 是否存在多策略共振

例如：

- S 级：高 TQS + 强催化 + bull regime + 非拥挤持仓
- A 级：高 TQS + 无明显负面新闻
- B 级：中等 TQS 或催化不够强
- C 级：边际合格，只允许小仓位

### 这类优化为什么重要

因为个人系统并不需要“平均对待所有机会”。  
真正提高收入期望的，是：

- 把更多风险预算给最值钱的少数机会
- 把边际机会缩成观察仓位

这会显著提高资本效率。

---

## 方向 D：优化“拒绝交易”的质量

### 核心思想

提高 alpha 不一定意味着“找到更多交易”，很多时候意味着：

- 更聪明地不做差交易

特别是个人系统，交易次数不是核心目标，**错误的高频是最贵的亏损来源**。

### 推荐优化问题

1. 哪类被接受的信号后续最差  
2. 哪类被 veto 的信号其实是好机会  
3. 哪类过滤器带来真正的净收益提升  
4. 哪类过滤器只是让系统显得更谨慎，但没有提高 EV  

### 为什么这很关键

如果系统能少做 20% 的低质量交易，且不伤害高质量机会，净收入期望通常比“多做 20% 平庸交易”更高。

---

## 方向 E：按市场状态做策略分化，而不是一套规则打天下

### 核心思想

同一套参数在：

- bull trend
- neutral chop
- bear rebound

下通常不是最优。

### 推荐做法

未来策略优化不应只问：

> 这个参数整体上有没有更好？

而应问：

> 它在什么市场状态下更好？

具体可以做：

- bull regime：更重趋势跟随、放宽利润奔跑空间
- neutral regime：提高质量门槛、降低仓位
- bear / shallow bear：只做特定板块或极高质量事件

### 为什么重要

alpha 不是静态的。  
一个真正赚钱的个人系统，必须让策略在不同市场状态下有不同的 aggressiveness。

---

## 五、最不应该做的优化方向

这部分同样重要，因为很多系统不是死于“没优化”，而是死于“优化错了”。

### 1. 不要继续大量增加新策略

当前系统已经不缺 entry 模式。  
再加第四、第五套入场策略，大概率只会：

- 增加复杂度
- 增加交互项
- 增加过拟合风险
- 稀释对现有高质量策略的理解

除非现有三类 alpha 已经被充分验证无法扩展，否则不建议优先加新策略。

### 2. 不要过度围绕少数样本修规则

例如：

- “这单亏了，所以加一个过滤器”
- “那单错过了，所以放松阈值”

这会快速把系统改成历史拟合机器。

### 3. 不要为了提高胜率而牺牲盈亏比

高胜率不是终点。  
一个 70% 胜率但偶尔大亏的系统，常常比一个 45%-55% 胜率、但大赢家保留得住的系统更差。

### 4. 不要让 LLM 接管硬风控

LLM 不应该决定：

- 精确止损位
- 精确仓位
- 组合热度
- 风险预算

这些必须继续由代码负责。

### 5. 禁止只增不减过滤器

> **2026-04-18 实测修正**：曾估算”11 层过滤器 × 70% 通过率 → 0.7^11 ≈ 2% 存活率”；
> 这是最坏情况理论估算，不是实测值。
> 实测存活率：**98%（50/51）**。
>
> 核心禁令不变：禁止盲目叠加过滤器，每层过滤器都必须能说清楚它过滤了什么、带来了多少误伤。
> 触发”禁止添加”的阈值应使用**实测值**，而非理论推算。
> 目标存活率 >5% 仍然有效；当前远超目标，增加过滤器的边际成本比过去更高。

### 6. 不要把”回测漂亮”当成”收入期望提升”的同义词

真正重要的是：

- 净收益
- 尾部风险
- 资本效率
- 多窗口稳定性
- 生产一致性

## 六、未来 90 天最值得做的 alpha 提升路线图

如果只允许用最少资源做最值钱的优化，我会建议按这个顺序推进。

### 路线图起点基线（2026-04-18）

| 指标 | 当前值 | 说明 |
|------|--------|------|
| `expected_value_score` | 0.4875 | 提升 >10% 才算有效改动；当前仅是观测样本 |
| Sharpe (daily) | 2.44 | 不应孤立地为 Sharpe 优化；应在不恶化尾部风险和资本效率的前提下改进 |
| Max Drawdown | 4.59% | 健康；守住 <10% 底线 |
| 交易次数（124天） | 30 笔 | trend:13, breakout:17, earnings:0 |
| 信号存活率 | 98% | 仅说明当前回测口径下后段过滤额外杀伤有限，不代表生产总过滤强度同样很低 |
| LLM 回放覆盖率 | **0%** | 最大测量盲区 |
| 新闻 veto 覆盖率 | 10.5% | 次要盲区 |
| 收敛状态 | CONVERGED（8/8） | 见 AGENTS.md §五 |

**关于 earnings_event_long = 0 笔交易：**

在 2025-10-20 → 2026-04-17 的回测窗口内，`earnings_event_long` 未产生任何成交。
当前**最强假设**是：回测器仅能重建 `days_to_earnings` 历史数据，无法重建
`positive_surprise_history` 和 `eps_estimate`（这些数据在历史上未持久化），从而让部分
本应成立的事件信号在回测口径下失去关键置信度加分。

但在没有信号级归因日志前，不应把这视为唯一根因。其他同样可能的原因包括：

- 事件窗口本身过窄
- 动量门槛过高
- ATR / RS / regime 过滤过严
- 与 A/B 策略竞争后未被选中
- 样本窗口本身缺少适合 C 策略的环境

这**不意味着策略逻辑有误**，而意味着：
1. 生产环境中该策略是否真实产生过信号，目前无法从回测数据验证；
2. 如需在回测中验证 C 策略，需要持久化每日 EPS surprise 快照；
3. 当前 EV score 完全来自 A+B 策略，earnings_event_long 仍是未知变量，而不是已被证伪的策略层。

建议第一阶段加入数据积累任务：开始记录每次 pipeline 运行时的 earnings 数据快照（`days_to_earnings`、`eps_estimate`、`surprise_history`）。

### 第一阶段：先提升评估质量

目标：让优化方向更可信

应做：

- 将 `expected_value_score` 作为正式主指标
- 在回测结果里固定输出资本效率、尾部损失、交易簇风险
- 对每次实验做结构化日志沉淀
- 明确 LLM veto / 放行后的代理效果

至少新增并长期追踪以下指标：

- `worst_trade_pct`
- `worst_3_trade_cluster_pct`
- `max_consecutive_losses`
- `tail_loss_share`
- `alpha_per_heat` 或其他单位风险预算收益代理指标

这一步不一定直接提高收益，但会减少大量无效优化。

### 第二阶段：集中优化 exit 体系

目标：保住大赢家，减少 alpha 泄漏

应做：

- 比较不同策略类型的 exit 表现
- 检查固定 target 是否过早切断趋势
- 检查 trailing 逻辑是否应更 regime-aware
- 用实验日志记录“exit 改动是否真的提升 EV”

### 第三阶段：让 LLM 输出结构化事件评分

**前置条件（未满足前禁止开始）：** `llm_prompt_resp_YYYYMMDD.json` 档案覆盖率 ≥ 30 天。

## 七、这套系统的最佳长期形态

如果这个个人系统未来发展成熟，我认为它的理想形态不是：

> “一个越来越复杂的规则集合”

而是：

> “一个以技术趋势为底座、以事件理解为增强、以仓位排序为核心、以实验记录为记忆体的个人资金分配系统”

它应当具备以下特点：

- `entry` 不求花哨，但求稳定
- `exit` 能保住真正的大赢家
- `LLM` 不接管风控，只增强事件理解和排序
- `position sizing` 有明显的质量分层
- `evaluation` 真正围绕赚钱期望而不是指标幻觉
- `logs` 能完整保留成功和失败实验

如果做到这一点，这个系统的 alpha 不一定来自“预测得更神”，而是来自：

- 更少做错
- 更久拿对
- 更大押在最值钱的机会

对于个人资金账户，这通常比“多找几个新策略”更能提高长期收入期望。

---

## 八、最后的指导原则

以后每次迭代前，都应该先问自己 5 个问题：

1. 这次优化是在提升 `entry` 质量，还是在提升 `exit` 质量？
2. 这次优化是在创造新 alpha，还是在减少已有 alpha 泄漏？
3. 这次优化是在提高资本效率，还是只是让系统看起来更复杂？
4. 这次优化是让 LLM 更可用，还是让 LLM 更不可控？
5. 这次优化最终会不会把风险预算更多地推向“最值得押的少数机会”？

如果一个改动回答不了这 5 个问题，就不应该优先做。

真正赚钱的个人量化系统，不是“什么都想做”，而是：

> **只把有限的资金、有限的注意力、有限的风险预算，压在最有期望值的少数机会之上。**
