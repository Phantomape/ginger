从最强大脑角度审计这个量化系统，目标是提升真实赚钱概率，而不是做表面优化。

要求：
1. 阅读整个代码仓库，梳理系统架构和策略执行链路。
2. 先输出问题审计报告，再进行任何修改。
3. 每个问题必须说明：
   - 所在位置
   - 为什么影响真实收益 / Sharpe / 回撤 / 成本 / 换手率
   - 证据是什么
   - 修复优先级
4. 优先修复高影响问题：数据泄漏、时间对齐错误、仓位 sizing、风控缺陷、成本建模缺失、指标计算 bug、LLM prompt 导致的过度交易或忽略风险约束。
5. 仅在不影响主目标的前提下，优化代码目录和可读性。
6. 所有改动必须先说明改动目的、影响范围和预期改善指标，再实施修改。
7. 必须运行测试和回测，并给出修改前后关键指标对比。
8. 如果结果没有改善，不要盲目尝试不同策略；先分析失败原因，形成新证据链后再进入下一轮。

禁止：
- 为了提高历史回测结果而过拟合
- 忽略手续费、滑点、流动性和执行约束
- 在没有证据的情况下大改策略
- 用代码重构掩盖策略无效
